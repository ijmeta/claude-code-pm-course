# Module 2: Advanced PM Work

**Duration:** ~1.5 hours | **Prerequisites:** Module 1 recommended

Real product management workflows applied to FinTrack. These modules mirror what PMs actually do day-to-day — and show you how Claude can accelerate each one.

---

## Modules

### 2.1 — Write a PRD
`/start-2-1` · 30 min

Learn how to write a great Product Requirements Document with Claude as your collaborator. Claude uses Socratic questioning to draw out your thinking, then helps you structure it into a full PRD.

**Files used:**
- `prd-templates/` — Three real-world PRD templates (Carl's, Joni's, Lenny's)
- `socratic-questioning.md` — The questioning framework Claude uses
- `user-research/pain-points.md` — FinTrack user pain points

**Output:** A complete PRD for a new FinTrack feature

---

### 2.2 — Analyze Data
`/start-2-2` · 25 min

The most comprehensive module. A full data-driven PM workflow in three phases:

**Phase 1 — Discovery:** Analyze activation funnel and survey data to identify where users are dropping off and why.

**Phase 2 — Estimation:** Build an ROI model and impact estimate for the problem you found.

**Phase 3 — Experiment Readout:** Analyze A/B test results with segmentation to make a ship/no-ship recommendation.

**Files used:**
- `activation-funnel-q4.csv` — Funnel drop-off data by step
- `fintrack-usage-data-q4.csv` — Feature usage and retention data
- `taskflow-usage-data-q4.csv` — 1,000 rows of task-level usage data
- `user-survey-responses.csv` — 800 user survey responses
- `onboarding-experiment-results.csv` — 8,000 rows of A/B test data
- `impact-estimation-framework.md` — ROI model template

**Output:** Problem analysis doc, impact estimate, experiment readout

---

### 2.3 — Product Strategy
`/start-2-3` · 30 min

Apply classic strategy frameworks to FinTrack's situation. Claude guides you through each framework and helps you produce a real strategy artifact.

**Files used:**
- `frameworks/gibson-biddle-dhm.md` — DHM model (Delight, Hard to Copy, Margin)
- `frameworks/rumelt-strategy-kernel.md` — Good Strategy / Bad Strategy kernel
- `frameworks/swot-analysis.md` — SWOT framework
- `methods/devils-advocate-strategy.md` — How to pressure-test your strategy

**Output:** A strategy document applying at least one framework to FinTrack

---

## What You'll Produce

- A complete PRD for a new FinTrack feature
- A data-driven problem analysis with funnel, survey, and experiment data
- An ROI/impact estimate
- An A/B test readout with a ship recommendation
- A product strategy document using a real framework
